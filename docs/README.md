# docs

该目录用于存放 LemonClaw 的项目文档。

## 目录结构
- `product/`：产品方案、PRD、功能范围、页面结构等文档
- `research/`：调研文档
- `architecture/`：架构设计、集成设计、数据结构说明
- `progress/`：项目进度、里程碑、TODO、阶段总结
- `daily/`：每日工作日志
- `decisions/`：关键产品或技术决策记录
- `templates/`：文档模板

## 使用建议
- 产品定义类文档放到 `product/`
- 调研结果放到 `research/`
- 偏实现和结构设计的内容放到 `architecture/`
- 每日日志建议使用日期命名，例如 `2026-04-20.md`
- 决策文档建议简短清晰，明确说明“为什么这样选”
